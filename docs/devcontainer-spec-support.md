# Support for Dev Container Specification

> Refer to the full Dev Container specification [here](https://containers.dev/implementors/json_reference/) for more information on the below options.

The symbols in the first column indicate the support status:

- 游릭 Fully supported.
- 游 Partially supported.
- 游댮 Not currently supported.

The last column indicates any currently existing GitHub issue for tracking support for this feature.
Feel free to [create a new issue](https://github.com/coder/envbuilder/issues/new) if you'd like Envbuilder to support a particular feature.

| Status                        | Name                                  | Description                                                                                                            | Known Issues                                           |
| ----------------------------- | ------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------ |
| **General**                   |                                       |                                                                                                                        |
| 游댮                            | `name`                                | Human-friendly name for the dev container.                                                                             | -                                                      |
| 游댮                            | `forwardPorts`                        | Port forwarding allows exposing container ports to the host, making services accessible.                               | [#48](https://github.com/coder/envbuilder/issues/48)   |
| 游댮                            | `portsAttributes`                     | Set port attributes for a `host:port`.                                                                                 | -                                                      |
| 游댮                            | `otherPortsAttributes`                | Other options for ports not configured using `portsAttributes`.                                                        | -                                                      |
| 游릭                            | `containerEnv`                        | Environment variables to set inside the container.                                                                     | -                                                      |
| 游릭                            | `remoteEnv`                           | Override environment variables for specific supporting services or tools, but not the container as a whole.            | -                                                      |
| 游릭                            | `remoteUser`                          | Override the user that supporting tools run as inside the container without changing the user for the whole container. | -                                                      |
| 游릭                            | `containerUser`                       | Override the user for all operations run inside the container.                                                         | -                                                      |
| 游댮                            | `updateRemoteUserUID`                 | Update the devcontainer UID/GID to match the local user.                                                               | -                                                      |
| 游댮                            | `userEnvProbe`                        | Shell to use when probing for user environment variables.                                                              | -                                                      |
| 游댮                            | `overrideCommand`                     | Override the default sleep command to be run by supporting services.                                                   | -                                                      |
| 游댮                            | `shutdownAction`                      | Action to take when supporting tools are closed or shut down.                                                          | -                                                      |
| 游댮                            | `init`                                | Adds a tiny init process to the container.                                                                             | [#221](https://github.com/coder/envbuilder/issues/221) |
| 游댮                            | `privileged`                          | Whether the container should be run in privileged mode.                                                                | -                                                      |
| 游댮                            | `capAdd`                              | Capabilities to add to the container (for example, `SYS_PTRACE`).                                                      | -                                                      |
| 游댮                            | `securityOpt`                         | Security options to add to the container (for example, `seccomp=unconfined`).                                          | -                                                      |
| 游댮                            | `mounts`                              | Add additional mounts to the container.                                                                                | [#220](https://github.com/coder/envbuilder/issues/220) |
| 游릭                            | `features`                            | Features to be added to the devcontainer.                                                                              | -                                                      |
| 游댮                            | `overrideFeatureInstallOrder`         | Override the order in which features should be installed.                                                              | [#226](https://github.com/coder/envbuilder/issues/226) |
| 游댮                            | `customizations`                      | Product-specific properties, e.g., _VS Code_ settings and extensions.                                                  | [#43](https://github.com/coder/envbuilder/issues/43)   |
| **Image/Dockerfile**          |                                       |                                                                                                                        |
| 游릭                            | `image`                               | Name of an image to run.                                                                                               | -                                                      |
| 游릭                            | `build.dockerfile`                    | Path to a Dockerfile to build relative to `devcontainer.json`.                                                         | -                                                      |
| 游릭                            | `build.context`                       | Path to the build context relative to `devcontainer.json`.                                                             | -                                                      |
| 游릭                            | `build.args`                          | Build args to use when building the Dockerfile.                                                                        | -                                                      |
| 游댮                            | `build.options`                       | Build options to pass to the Docker daemon. Envbuilder does not use a Docker daemon, so this is not relevant.          | -                                                      |
| 游릭                            | `build.target`                        | Target to be passed when building the Dockerfile.                                                                      | -                                                      |
| 游릭                            | `build.cacheFrom`                     | Images to use as caches when building the Dockerfile.                                                                  | -                                                      |
| 游댮                            | `appPort`                             | Ports to be published locally when the container is running.                                                           | -                                                      |
| 游댮                            | `workspaceMount`                      | Overrides the default local mount point for the workspace when the container is created.                               | -                                                      |
| 游댮                            | `workspaceFolder`                     | Default path to open when connecting to the container.                                                                 | -                                                      |
| **Docker Compose**            |                                       |                                                                                                                        |
| 游댮                            | `dockerComposeFile`                   | Path to a Docker Compose file related to the `devcontainer.json`.                                                      | [#236](https://github.com/coder/envbuilder/issues/236) |
| 游댮                            | `service`                             | Name of the Docker Compose service to which supporting tools should connect.                                           | [#236](https://github.com/coder/envbuilder/issues/236) |
| 游댮                            | `runServices`                         | Docker Compose services to automatically start.                                                                        | [#236](https://github.com/coder/envbuilder/issues/236) |
| **Lifecycle scripts**         |                                       |                                                                                                                        |
| 游댮                            | `initializeCommand`                   | Command to run on the host machine when creating the container.                                                        | [#395](https://github.com/coder/envbuilder/issues/395) |

| 游릭                            | `postStartCommand`                    | Command to run each time the container is started.                                                                     |                                                        |
| 游댮                            | `postAttachCommand`                   | Command to run each time a tool attaches to the container.                                                             |                                                        |
| 游댮                            | `waitFor`                             | Specify the lifecycle command tools should wait to complete before connecting.                                         |                                                        |
| **Minimum Host Requirements** |                                       |                                                                                                                        |

| 游댮                            | `hostRequirements.memory`             | Minimum memory requirements.                                                                                           | -                                                      |

| 游릭                            | `${localWorkspaceFolder}`             | Path to the local workspace folder.                                                                                    | -                                                      |

| 游댮                            | `${devcontainerId}`                   | A stable unique identifier for the devcontainer.                                                                       | -                                                      |
